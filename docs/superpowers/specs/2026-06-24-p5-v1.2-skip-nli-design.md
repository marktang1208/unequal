# P5 v1.2 NLI 降级触发条件 — 短答案跳过 NLI 设计

> 日期: 2026-06-24
> 前置: [state-p5-v1.1.1.md](./state-p5-v1.1.1.md) — 真接发现 3 bug 已修 + 网络限制确认
> 状态: ✅ 设计完成,等用户 review → 写 implementation plan

## 1. 目标

P5 v1.1 真接 production 发现 CloudBase → 硅基流动 网络 90% 调用 15s 内返不了,
NLI 后置验证在生产命中率 0-20%,且 LLM prompt 兜底已拦截大部分超知识库问题。
P5 v1.2 **降级 NLI 触发条件**:只对 LLM answer 字符数 >= 阈值的回答调 NLI,
短答案(没空间塞幻觉)直接跳过,节省延迟 + 不浪费调用。

## 2. 设计

### 2.1 阈值

- **默认阈值**:100 字符(`cleaned` 长度,去除 [N] 引用后)
- **可配置**:`NLI_MIN_ANSWER_LEN` env
- **判断**: `cleaned.length < NLI_MIN_ANSWER_LEN` 跳过 NLI

### 2.2 跳过行为

- 不调 `getNliProvider().verify()`
- 不写 audit(`ask_nli_reject` 不写,因为没调)
- `verdict` 视为 entailed(无 warning prefix)
- 响应 answer 字段保留原文 [N](用户端体验不变)
- 日志: `[nli] skipped: answer too short (N < threshold)`

### 2.3 数据流

```
ask handler (api-ask.ts)
  ↓
LLM 生成 answer → parseAnswerSegments → cleaned
  ↓
cleaned.length < NLI_MIN_ANSWER_LEN ?
   ├─ YES: 跳过 NLI,verdict = entailed,无 audit
   └─ NO:  现有 NLI 路径(调 verify → 写 audit reject 或 pass)
```

### 2.4 改动清单

| 文件 | 改动 |
|---|---|
| `apps/api/src/handlers/api-ask.ts` | 第 7 步 NLI 调用前加长度判断 + 跳过分支(预计 +12 行) |

| `apps/api/src/lib/nli/__tests__/get-provider.test.ts` | 加 1 个 test:`env NLI_MIN_ANSWER_LEN` 读取(覆盖 env 读取链路) |

### 2.5 不动

- `http-provider.ts` / `get-provider.ts` / `apply-warning.ts` 不动
- `api-chat.ts` 不动(chat 路径 NLI 是 v2 候选,本次不涉及)
- LLM prompt 兜底逻辑不动

## 3. 边界

| 维度 | 处理 |
|---|---|
| **阈值比较** | `< 100` 跳过,`>= 100` 调 NLI |
| **长度指代** | `cleaned`(去 [N] 引用后的纯答案),非 `answer` 原文 |
| **audit 行为** | 跳过场景**不写** audit(节省存储) |
| **response 行为** | 与现在一致(无 warning prefix,answer 字段保留原文) |
| **降级触发** | 仅按长度,**不**叠加 retrieval 质量等其它判断 |
| **provider 行为** | 跳过不触发 `getNliProvider()`,所以 5min cache / 10-timeout 状态机不变 |
| **error path** | 长度判断是纯字符串比较,不可能 fail;不引入新错误路径 |
| **并发** | 单请求内同步判断,无并发问题 |

## 5. 风险与回滚

| 风险 | 缓解 | 回滚 |
|---|---|---|
| 阈值过低导致漏判短答案 hallucination | LLM prompt 兜底(规格强制知识库无答案明确说明) | `NLI_MIN_ANSWER_LEN=0` 即恢复 v1.1 行为 |
| 用户预期短答案也要 NLI 验证 | 真接监控跳过率 | 调低阈值或改默认 |
| audit 数据缺失影响后期分析 | 长答案仍全量 audit | 短答案本就没必要 NLI |
| env 变量没生效 | 加 unit test 验证读取 + 真接看日志 | 同 NLI_TIMEOUT_MS 的修复路径(env 默认 + opts override) |
