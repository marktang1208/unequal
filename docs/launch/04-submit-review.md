# 提交审核操作步骤

> 前提: 已完成 `03-real-device-test.md` 全部 5 路径 ✅, 已按 `01-submission-materials.md` 准备物料, 已按 `02-github-pages-setup.md` 部署协议页。
> 预计耗时: 30 分钟。

---

## 1. 微信开发者工具上传代码 (5 分钟)

### 1.1 编译

1. 微信开发者工具 → 顶部菜单「工具」→ 构建 npm (如果之前没构建)
2. 等待编译完成 (右下角进度条 100%)
3. 顶部菜单「工具」→ 清缓存 → 全选 → 清空

### 1.2 上传

1. 右上角「上传」按钮
2. 弹窗填写:

| 字段 | 值 |
|---|---|
| 版本号 | `1.0.0` |
| 项目备注 | `首发版本: 育儿知识问答, 引用追溯, P8 + P9 真接` |

3. 点「上传」
4. 进度条走完 → 看到「上传成功」提示

### 1.3 失败信号

| 现象 | 排查 |
|---|---|
| 上传按钮灰 | 还没编译完, 等 100% |
| 报"代码包过大" | 主包 > 2MB, 看 miniprogramRoot, 删不用的图 |
| 报"AppID 不匹配" | project.config.json appid 改错, 改回 `wxf5b8ce05a977f0c6` |

---

## 2. 微信公众平台 → 设为体验版 (3 分钟)

### 2.1 入口

打开浏览器: https://mp.weixin.qq.com

登录: 扫码登录 (用绑定该 AppID 的微信)

### 2.2 路径

左侧菜单 → 「管理」→ 「版本管理」

### 2.3 设为体验版

1. 在「开发版本」列表找到刚才上传的 `1.0.0`
2. 点右侧「设为体验版」
3. 弹窗确认 → 点「确定」
4. 体验版二维码生成 → **用另一个微信号扫码**测试

> ⚠️ 必须用**非管理员的微信号**扫码测试, 否则部分功能 (获取手机号等) 看不到真实行为。

### 2.4 体验版测试

按 `03-real-device-test.md` 再走一遍 5 路径 (体验版的代码包就是审核版, 必须再验一次)。

---

## 3. 提交审核 (10 分钟)

### 3.1 入口

「版本管理」→ 体验版列表 → 右侧「提交审核」

### 3.2 填写审核信息

#### 3.2.1 必填项

| 字段 | 值 |
|---|---|
| 服务类目 | **生活服务 → 母婴** (一级: 生活服务, 二级: 母婴) |
| 标签 | 育儿, 母婴, 辅食, 早教, 问答 (从 §3 选) |
| 简介 | 从 §2 选 A/B/C (推荐 A) |
| 页面截图 | 5 张 (从 §5 准备) |

#### 3.2.2 体验版二维码

- 自动生成, 不需要手动填

#### 3.2.3 测试账号

- 个人主体无需登录即可使用 → 填「无」或留空

#### 3.2.4 备注 (重要!)

> 本小程序为家长提供育儿问答服务, 基于用户上传/抓取的个人育儿知识库进行检索增强生成, 每条回答附带原文引用卡片。
>
> **核心功能**:
> 1. 一对一问答（chat 页, 含引用卡片）
> 2. 历史问答记录
> 3. 引用原文详情查看
> 4. 用户设置（user_id 显示, 来源类型过滤）
>
> **技术**: 微信云开发 (CloudBase) + 微信小程序, 不涉及支付/位置/通讯录/广告, 不收集个人敏感信息。
>
> **个人主体可用类目**: 生活服务 - 母婴。

### 3.3 协议 + 政策 URL

| 字段 | URL |
|---|---|
| 用户协议 | `https://YOUR_GITHUB_USERNAME.github.io/unequal-legal/` |
| 隐私政策 | `https://YOUR_GITHUB_USERNAME.github.io/unequal-legal/privacy.html` |

> ⚠️ 这两个 URL 必须是**公开可访问的 HTTPS URL**。如果没部署, 审核会被打回。

### 3.4 提交

点「提交审核」按钮 → 弹窗确认 → 「确定」

---

## 4. 审核等待 (1-3 天)

### 4.1 状态查询

入口: 「版本管理」→ 「审核版本」

| 状态 | 含义 |
|---|---|
| 审核中 | 排队等审核员 (个人主体一般 24h 内) |
| 审核通过 | 出现「发布」按钮 |
| 审核不通过 | 出现「重新提交」+ 不通过原因 |

### 4.2 期间可以做

- 加真育儿 corpus (上线后并行)
- 准备 P10 (NLI cold-start race)
- 监控 audit_log, 提前发现潜在问题

---

## 5. 审核通过 → 发布 (5 分钟)

### 5.1 发布

「版本管理」→ 「审核版本」→ 点「发布」

弹窗:
- 全量发布: 立即所有用户可见
- 灰度发布: 选 5% / 10% / 30% / 50% (推荐先用灰度)

> **建议先灰度 5%** (1-2 天), 没大问题再 100%。但个人主体没这个选项, 默认全量。

### 5.2 发布后

- 小程序正式上线
- 微信搜索「育儿不等号」能找到
- 用户可以扫码 / 搜索使用

---

## 6. 发布后 7 天监控 (上线后每日)

| 指标 | 目标 | 怎么查 |
|---|---|---|
| 启动次数 | 持续增长 | 微信公众平台 → 统计 |
| 崩溃率 | < 1% | 微信公众平台 → 运维中心 |
| API 错误率 | < 5% | CloudBase 控制台 → 云函数日志 |
| 引用命中数 | 持续增长 | CloudBase 控制台 → D1 SQL |
| 「未涉及」率 | < 50% | 抽样查 audit_log |

---

## 7. 常见审核不通过原因 + 应对

| 原因 | 应对 |
|---|---|
| 类目不对 | 改类目到「生活服务 - 母婴」 |
| 协议 URL 不可访问 | 重新部署 GitHub Pages, 验公开 |
| 简介超过 50 字 | 缩短到 30 字内 |
| 截图包含个人信息 | 重新截图, 隐藏昵称/头像 |
| 类目资质缺失 | 个人主体 + 母婴不要资质, 备注说明 |
| 测试账号不可用 | 填「无需登录」 |
| 体验版功能异常 | 修复后重新上传 |
