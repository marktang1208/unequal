# Plan: M6.2 wx.login + JWT + admin 登录页

- **Spec**：`docs/superpowers/specs/2026-06-16-m6-2-jwt-auth-design.md`
- **日期**：2026-06-16
- **复杂度**：Medium（4 包协同 + 鉴权重构）
- **Mock-first 边界**：jscode2session 走 fetchImpl mock；jose 真跑（HS256 同步）；不真接 Cloudflare / 微信真机

---

## 1. Requirements Restatement

把 M6.1 的 `admin_token` 单模式升级到「admin_token + wx.login」双模式，鉴权走统一 `verifyAuth` 入口（`AUTH_MODE` 切）。

**核心交付**：
- `/auth/wx-login` + `/auth/admin-login` 2 个 endpoint
- `verifyAuth` 加 `jwt` 分支（替换 M6.1 的 501 留口）
- 6 个现有路由从 `verifyAdminToken` 改 `verifyAuth`
- `lib/auth-jwt.ts`（jose 签发 + 验签）
- `lib/wx.ts`（jscode2session fetchImpl 包装）
- `lib/user.ts`（findOrCreateUser upsert D1）
- admin LoginPage + 小程序冷启动 `wx.login`
- ~30 新增 vitest 用例

**不交付**（推到 M6.3+）：session_key 存储 / refresh token / 多租户 / admin rate limit

## 6. Risks

| 风险 | Likelihood | Mitigation |
|---|---|---|
| jose dep 体积 (~50KB) | LOW | 1 个 lib 1 个 dep；admin 不直接 import jose（admin 走 jose 验签在后端）|
| JWT_SECRET 泄露 → 任意伪造用户 | LOW | wrangler secret 定期 rotate（季度）；不写代码；wrangler secret 历史不进 git |
| jscode2session 限频（5 万次/分钟/AppID） | LOW | M6.2 用户量小，< 1k req/min；超限时 5 分钟重试 |
| wx.login code 5 分钟过期 nonce 重放 | LOW | M6.2 不防重放（不存 session_key）；M6.3+ 加 nonce store |
| admin 登录页不防爆破 | LOW | env.ADMIN_TOKEN 强密码（32+ 字节随机）足够；M6.3+ 加 rate limit |
| wx.login 失败阻断 app 启动 | LOW | app.ts onLaunch 失败不 throw（仅 console.warn），401 时 chat() retry |
| verifyAuth 重构破现有 6 路由 | MEDIUM | 路由层 verifyAdminToken → verifyAuth 是直接替换；CP-2 跑全 api test 验证 |
| miniflare 测试 jose 异步签名时序 | LOW | jose HS256 是同步 Web Crypto API（node + workerd 都支持），无需 mock |
| subagent 跑 Phase 3+4 (admin + miniprogram) 复杂调试 stall | MEDIUM | 每 subagent 1-2 task + heartbeat + 主线程 5-10 min git log 查进展（feedback_subagent_heartbeat_monitoring）|
| admin LoginPage navigate guard 跟 React Router 5 兼容 | LOW | `react-router-dom` v5 `useNavigate` hook；admin 用 v5 (M3 已装) |
