## 目标

让 NLI 在 LLM answer < 100 字符时跳过调用,降低 ask 平均延迟。
实现路径:`api-ask.ts` 加长度判断 + `cloudbaserc.json` 加 env + 新 unit test。

## 实施步骤

### Step 1: 加 unit test(RED)

**文件**: `apps/api/src/handlers/__tests__/api-ask.skip-nli.test.ts`(新)

**目的**:先写测试,锁定期望行为。

**测试内容**:
- `cleaned` 长度 < 100(默认) → `getNliProvider()` 不被调用
- `cleaned` 长度 >= 100 → `getNliProvider()` 被调用
- env `NLI_MIN_ANSWER_LEN=200` → 阈值改为 200

**为什么先写测试**: TDD 纪律,避免实现完才发现行为不对。
