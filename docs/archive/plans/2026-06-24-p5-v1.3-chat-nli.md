/**
 * P5 v1.3: 抽 session 持久化为 helper,skip path 与 NLI path 共用
 * @param session 当前 session(已含 newMessages)
 * @param isNewSession 是否新 session
 */
async function persistSessionMessages(
  session: ChatSession,
  newMessages: ChatMessage[],
  title: string,
  isNewSession: boolean,
  now: number,
): Promise<void> {
  if (isNewSession) {
    await add<ChatSession>(COLLECTIONS.chatSession, {
      ...session,
      title,
      messages: newMessages,
      updatedAt: now,
    });
  } else {
    const { update } = await import("../lib/db.js");
    await update(COLLECTIONS.chatSession, session.id, {
      title,
      messages: newMessages,
      updatedAt: now,
    });
  }
}

**实现要点**:
- **二次持久化**:NLI reject 路径需要把 finalAnswer 覆盖原 answer 持久化。session.messages 在 helper 内已被 update,所以**二次** update(从数据库读最新 → 替换最后一条 → 再 update)— OR — **提前** save 一个临时数据结构,在 NLI 完成后整体持久化

**简化方案**(推荐):把"持久化"挪到 NLI 之后,**整段流程**是:
```
1. LLM chat → answer
2. parseAnswerSegments(answer) → citedNums, cleaned
3. shouldSkipNli(cleaned) → yes → 持久化 raw answer + 返
4. NLI verify → verdict (or fail-open)
5. audit (if reject)
6. finalAnswer = applyWarning(answer, verdict) (or fail-open 不变)
7. 持久化 finalAnswer (含 [N] 引用)
8. 返 finalAnswer + citedNums + citations
```

这样**只持久化一次**,skip path 和 NLI path 行为一致。
